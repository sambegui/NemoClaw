LangChain Deep Agents Code support is experimental.
Use it when you want NemoClaw to build an OpenShell sandbox with the `dcode` terminal coding agent installed and configured for NemoClaw-managed inference.

## Onboard

Run onboarding with the canonical agent ID.

```bash
nemoclaw onboard --agent langchain-deepagents-code
```

The image installs a pinned Deep Agents Code release with NVIDIA provider support.
NemoClaw writes `/sandbox/.deepagents/config.toml` with an OpenAI-compatible provider pointed at `https://inference.local/v1`, uses a scoped placeholder API key for that managed route, and sets `use_responses_api = false` for Chat Completions compatibility.
Real provider credentials remain in NemoClaw/OpenShell credential handling and are not written into the Deep Agents config file.

## Use the Harness

Connect to the sandbox, then launch the terminal UI.

```bash
nemoclaw <sandbox-name> connect
dcode
```

For a single headless task, run:

```bash
dcode -n "Summarize this repository"
```

The managed wrapper launches Deep Agents Code with `HOME=/sandbox`, update checks disabled, remote Deep Agents sandbox providers disabled, and MCP auto-loading disabled.
Non-interactive shell execution remains disabled unless you explicitly set a NemoClaw-approved allow list.

## State and Backup

Deep Agents Code state lives under `/sandbox/.deepagents`.
NemoClaw snapshot and rebuild flows preserve the app state directory, skills, generated config, hooks config, and MCP config when those files exist.
The `.env` file is intentionally not preserved because users may put Tavily, LangSmith, or provider credentials there.

## Optional Web Search

Deep Agents Code can use Tavily web search when a Tavily credential is provided in the runtime environment.
NemoClaw does not enable Tavily by default for this experimental harness.

## Troubleshooting

Use normal sandbox lifecycle commands:

```bash
nemoclaw <sandbox-name> status
nemoclaw <sandbox-name> logs --follow
nemoclaw <sandbox-name> rebuild
nemoclaw <sandbox-name> snapshot create --name before-change
```

`status` reports the selected harness as a terminal runtime and prints the interactive/headless command shape.
There is no dashboard port or long-running gateway process for this harness.
